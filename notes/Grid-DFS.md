# Grid DFS 笔记

## 1. 这类题到底在做什么

这类题的本质通常不是“遍历二维数组”，而是：

- 在网格里找一整块连通区域
- 用 DFS / BFS 一次性把这一整块处理完

常见特征：

- 题目给你一个 `grid` / `board`
- 只能往上、下、左、右走
- 某种字符或数字表示“目标格子”
- 需要统计、染色、保护、求面积

一句话理解：

> 先找“从哪里开始扩散”，再让 `helper` 负责“把这一整块连通区域处理完”。

---

## 2. 通用思考顺序

每次做网格 DFS，先问自己 4 个问题：

1. 目标格子是谁？
   - 例如 `numIslands` 里是 `'1'`
   - `solve` 里是 `'O'`

2. 从哪里启动 DFS？
   - 是全图扫描到目标就启动
   - 还是只从边界启动

3. 访问过怎么标记？
   - 改成 `'0'`
   - 改成 `'P'`
   - 或者用 `visited`

4. 最后要什么结果？
   - 数量
   - 面积
   - 保留 / 清除

---

## 3. 通用 DFS 模板

```java
void dfs(char[][] board, int i, int j) {
    if (i < 0 || i >= board.length || j < 0 || j >= board[i].length) return;
    if (board[i][j] != 'O') return;

    board[i][j] = 'P';

    dfs(board, i - 1, j);
    dfs(board, i + 1, j);
    dfs(board, i, j - 1);
    dfs(board, i, j + 1);
}
```

这个模板的职责非常单一：

- 越界就停
- 不是目标就停
- 标记当前格子
- 向四个方向扩散

注意：

- `helper` 只负责“处理整块区域”
- 不负责“决定从哪里开始”

“从哪里开始调用 `helper`” 这件事应该放在主函数里想。

---

## 4. 三道题对比

| 题目 | 目标格子 | 启动 DFS 的时机 | `helper` 做什么 | 结果怎么得出 |
| --- | --- | --- | --- | --- |
| `numIslands` | `'1'` | 扫描全图，遇到 `'1'` 就启动 | 把整片岛屿都淹掉 | 启动一次，答案加一 |
| `solve` | `'O'` | 只从边界上的 `'O'` 启动 | 把和边界连通的 `O` 标成安全 | 最后把剩余 `O -> X`，`P -> O` |
| `maxAreaOfIsland` | `1` | 扫描全图，遇到 `1` 就启动 | 统计当前连通块面积并标记已访问 | 取最大面积 |

---

## 5. `numIslands` 的核心理解

题目本质：

- `1` 是陆地
- 上下左右连起来的一整片 `1` 算一个岛

主函数思路：

- 扫描整张图
- 每次遇到一个还没处理过的 `'1'`
- 说明发现了一座新岛
- 答案加一
- 然后用 DFS 把这一整座岛全部改掉

关键点：

- 为什么可以直接改成 `'0'`
  - 因为你只是为了避免重复统计

一句话记忆：

> 发现一个新岛，就顺手把整座岛淹掉。

---

## 6. `solve` 的核心理解

题目本质：

- 只有那些“完全被包围”的 `O` 才能变成 `X`
- 只要某个 `O` 能通过上下左右一路连到边界，它就不能被改掉

这题最容易想反。

错误思路：

- 找哪些 `O` 应该变成 `X`

正确思路：

- 先找哪些 `O` 绝对不能变成 `X`

主函数流程：

1. 扫描四条边
2. 边界上如果有 `O`，就启动 DFS
3. 把所有和边界连通的 `O` 标成 `P`
4. 再扫全图
5. 把剩余 `O -> X`
6. 把 `P -> O`

一句话记忆：

> 这题不是主动找“该删谁”，而是先保护“绝对不能删的人”。

---

## 7. `maxAreaOfIsland` 怎么套模板

和 `numIslands` 很像，区别只是：

- `numIslands` 关心有几块
- `maxAreaOfIsland` 关心某一块有多大

所以变化点只有一个：

- `helper` 不只是标记访问过
- 还要返回当前这一整块的面积

一句话记忆：

> `numIslands` 是“数块数”，`maxAreaOfIsland` 是“算块大小”。

---

## 8. 最容易犯的坑

### 8.1 把“连通区域”想成“矩形区域”

错误表现：

- 从某个点开始，直接用 `for` 扫一整行或一整列

为什么错：

- DFS 处理的是连通性
- 不是几何范围
- 安全区域、岛屿区域的形状通常是不规则的

正确做法：

- 只处理当前点
- 然后递归到上下左右

### 8.2 `helper` 里塞太多逻辑

错误表现：

- 在 `helper` 里判断“是不是边界启动点”
- 在 `helper` 里写一堆业务判断

正确分工：

- 主函数：决定从哪里开始 DFS
- `helper`：处理从这个点出发的一整块连通区域

### 8.3 越界条件写错

常见正确写法：

```java
if (i < 0 || i >= board.length || j < 0 || j >= board[i].length) return;
```

容易错成：

```java
j > board[i].length
```

实际上只要 `j == board[i].length` 就已经越界了。

### 8.4 字符写错

这类题里常见的是：

- `'O'` 是字母 O
- `'0'` 是数字 0

这两个非常容易混。

---

## 9. 做题时的口头模板

看到网格 DFS 题时，可以先在脑子里默念：

1. 目标格子是谁？
2. 从哪里开始搜？
3. 访问过以后怎么标记？
4. 最后统计什么？

如果这四个问题能回答清楚，代码通常就不难写了。

---

## 10. 一页速记

### `numIslands`

- 目标格子：`'1'`
- 启动点：全图中每个还没访问过的 `'1'`
- 操作：把整片 `'1'` 变成 `'0'`
- 结果：启动几次 DFS 就有几个岛

### `solve`

- 目标格子：`'O'`
- 启动点：边界上的 `'O'`
- 操作：把边界连通的 `'O'` 标成 `P`
- 结果：`O -> X`，`P -> O`

### `maxAreaOfIsland`

- 目标格子：`1`
- 启动点：全图中每个还没访问过的 `1`
- 操作：DFS 统计当前连通块面积
- 结果：取最大值

---

## 11. 最后一句总结

网格 DFS 题不神奇，真正关键的是两件事：

- 你能不能看出“这题在处理连通块”
- 你能不能想清楚“DFS 从哪里开始启动”

只要这两件事想明白，`helper` 往往都会变得很短。
